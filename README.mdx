# Pre-Internship ToDo Course: Guide

## Recommended Order for Completing Tasks

1. **JS-TS**: Start with the JavaScript/TypeScript tasks. These cover basic data structures, functions, and ToDo logic. Complete all 10 tasks in the `JS-TS/tasks` folder.
2. **React-CSS**: Move to the React tasks. Build UI components for the ToDo app, manage state, handle events, and style your components. Complete all 10 tasks in the `React-CSS/tasks` folder.
3. **Node-Express-Nest**: Next, work on backend tasks. Start with Express.js tasks, then move to NestJS. Implement API endpoints, middleware, DTOs, and services for the ToDo app. Complete all 10 tasks in the `Node-Express-Nest/tasks` folder.
4. **DB-NoSQL**: Practice SQL and NoSQL queries for ToDo data. Complete all 10 tasks in the `DB-NoSQL/tasks` folder.
5. **Docker-Git**: Learn how to containerize your app and manage it with Git. Complete all 10 tasks in the `Docker-Git/tasks` folder.

## How to Work with This Repository

1. **Fork This Repository**
   - Click the "Fork" button on GitHub to create your own copy of this repository.
2. **Clone Your Fork**
   - Clone your forked repository to your local machine:
     ```sh
     git clone https://github.com/YOUR_USERNAME/YOUR_FORK.git
     ```
3. **Work Locally**
   - Complete tasks in the appropriate folders. You can work on tasks in order or in parallel, but following the recommended order is best for learning.
   - Use the provided `solutions` folders to implement your answers. Do not modify the `tasks` files directly.
   - Use the provided test files to check your solutions.
4. **Commit and Push**
   - Commit your changes regularly:
     ```sh
     git add .
     git commit -m "Complete JS-TS tasks 1-3"
     git push origin main
     ```
   - Push your solutions and all related files (including your code, tests, and configs) to your forked repository.
5. **Share Your Repository**
   - When you are ready for review, share the link to your forked repository with your mentor or reviewer.

## Tips
- Keep your solutions organized in the `solutions` folders.
- Do not push `node_modules`, `.env`, or other files listed in `.gitignore`.
- Use branches if you want to experiment or work on features separately.
- Read the description in each `.mdx` task file carefully before starting.

Good luck and happy coding!
